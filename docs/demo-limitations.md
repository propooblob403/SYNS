# Demo Limitations

This repository is a protocol-facing MVP, not a live intelligence network.

## What is real in the repo

- typed domain models
- deterministic scenarios
- reproducible emergence runs
- public demo endpoints
- an interactive lab surface

## What is not yet real

- live onchain ingestion
- social firehose integration
- distributed agent execution
- historical memory persistence
- production scoring calibration

The purpose of the MVP is to prove the architecture direction without pretending that the final system already exists.
